# Plan: Calendar View for Todo Manager

> Created: 2026-04-05

## Executive Summary

| 관점 | 설명 |
|------|------|
| **Problem** | 할 일에 마감일이 없어 시간 관리가 불가능하고, 전체 일정을 한눈에 파악할 수 없음 |
| **Solution** | 각 task에 target date/time 필드 추가 + 월/주/일 캘린더 뷰 + 캘린더 내 풀 편집 |
| **Function UX Effect** | 기존 리스트 뷰와 캘린더 뷰를 탭으로 전환하며 시간 기반으로 할 일을 시각적으로 관리 |
| **Core Value** | 할 일의 시간적 맥락을 파악하여 우선순위와 마감일을 효과적으로 관리 |

| 항목 | 내용 |

| **SUCCESS** | 월/주/일 뷰 전환 가능, 캘린더에서 task CRUD + 드래그 이동, 반응형 동작 |

## 1. 배경

현재 Todo Manager는 리스트 기반으로 할 일을 관리한다. 하지만:
- task에 **마감일/시간** 개념이 없어서 "이번 주 금요일까지" 같은 시간 관리 불가
- 전체 일정을 **시각적으로 한눈에** 볼 수 없음
- 시험 날짜, 숙제 마감일 등 **날짜 기반 task**를 효과적으로 관리할 수 없음

## 2. 요구사항

### 2.1 핵심 요구사항 (P0)

| ID | 요구사항 | 설명 |
|----|----------|------|
| R1 | Task에 target date/time 필드 추가 | 선택적 날짜+시간 입력. 없으면 "No date" |
| R2 | 리스트/캘린더 뷰 전환 | 상단 탭 네비게이션으로 두 뷰 전환 |
| R3 | 월간 뷰 (Month) | 달력 그리드에 각 날짜의 task 표시 |
| R4 | 주간 뷰 (Week) | 7일 칼럼 + 시간 행 그리드 |
| R5 | 일간 뷰 (Day) | 시간 단위 타임라인 (0시~23시) |
| R6 | 캘린더 내 task 생성 | 날짜/시간 셀 클릭 → 인라인 task 생성 |
| R7 | 캘린더 내 task 수정/삭제 | task 클릭 → 팝업/인라인 편집 |
| R8 | 반응형 레이아웃 | 데스크톱: 넓은 캘린더, 모바일: 축소 적응 |
| R9 | 데이터 영속성 | 날짜 데이터 포함 localStorage 저장 |

### 2.2 보조 요구사항 (P1)

|----|----------|------|

### 4.2 뷰별 상세

**월간 뷰 (Month)**
- 7열 x 5~6행 그리드
- 각 셀에 날짜 번호 + task 최대 2~3개 표시 ("+N more" 링크)
- 오늘 날짜 셀 파란색 원 강조
- 빈 셀 클릭 → 해당 날짜로 task 생성
- task 드래그 → 다른 날짜로 이동

**주간 뷰 (Week)**
- 7일 칼럼, 왼쪽에 시간 행 (0:00~23:00)
- 각 task가 시간 범위에 맞게 블록으로 표시
- dueTime 없는 task는 상단 "All day" 영역에 표시

**일간 뷰 (Day)**
- 세로 시간 타임라인 (0:00~23:00, 1시간 단위)
- task가 해당 시간 위치에 블록으로 표시
- 시간 슬롯 클릭 → 해당 시간에 task 생성
- dueTime 없는 task는 상단 "All day" 영역에 표시

### 4.3 Task 입력 변경
기존 입력 영역에 날짜/시간 선택 추가:
```
[ 할 일 입력... ] [날짜 📅] [시간 🕐] [카테고리 ▼] [추가]
```
- 날짜: `<input type="date">`
- 시간: `<input type="time">`
- 둘 다 선택적 (비워두면 "No date")

## 5. 기술 설계 방향

### 5.1 뷰 전환 방식
- 사이드바에 List/Calendar 탭 추가
- 상태 변수 `activeView`로 전환
- 각 뷰는 같은 `main-content` 내에서 show/hide

### 5.2 캘린더 렌더링
- 순수 JS + DOM 조작으로 그리드 생성
- `Date` 객체로 달력 계산 (윤년, 월별 일수 등)
- CSS Grid로 7열 레이아웃

### 5.3 드래그 앤 드롭
- HTML5 Drag and Drop API (dragstart, dragover, drop)
- 모바일: touch 이벤트로 폴백

### 5.4 변경 파일
| 파일 | 변경 내용 |
|------|-----------|
| `index.html` | 캘린더 뷰 컨테이너, 날짜/시간 입력 필드, 뷰 탭 추가 |
| `style.css` | 캘린더 그리드, 뷰 전환, 반응형 스타일 추가 |
| `script.js` | 캘린더 렌더링 로직, 뷰 전환, 날짜 CRUD, 드래그 앤 드롭 |

## 7. 리스크

| 리스크 | 영향 | 완화 방안 |
|--------|------|-----------|
| 순수 JS 캘린더 복잡도 | 높음 | 모듈화하여 month/week/day 렌더러 분리 |
| 모바일 드래그 앤 드롭 | 중간 | touch 이벤트 폴백, 모바일에서는 클릭 편집 우선 |

| 기존 task 호환성 | 낮음 | dueDate/dueTime null 처리, 기존 데이터 자동 호환 |
