# PRD: 개인용 할 일 관리 앱 (Todo Manager)

> Created: 2026-04-05

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **제품명** | Todo Manager |
| **대상 사용자** | 매일 10~20개의 할 일을 관리하는 개인 사용자 |
| **플랫폼** | 웹 브라우저 (데스크톱/모바일) |
| **기술 스택** | HTML5 + CSS3 + Vanilla JavaScript (프레임워크 없음) |
| **데이터 저장** | localStorage (서버 없음, 오프라인 동작) |

## 2. 문제 정의

### 2.1 해결하려는 문제
일상적인 할 일(업무, 개인, 공부)이 여러 곳에 흩어져 있어 관리가 어렵다.
무거운 앱을 설치하거나 계정을 만들 필요 없이, 브라우저를 열자마자 바로 사용할 수 있는 가벼운 할 일 관리 도구가 필요하다.

### 2.2 핵심 가치
- **즉시 사용**: 설치/가입 없이 브라우저에서 바로 사용
- **데이터 영속성**: 새로고침/브라우저 재시작 후에도 데이터 유지
- **카테고리 관리**: 업무/개인/공부를 한눈에 분류하여 확인
- **진행률 시각화**: 오늘의 달성률을 직관적으로 파악

## 3. 사용자 페르소나

| 항목 | 내용 |
|------|------|
| **이름** | 김주호 (직장인/학습자) |
| **행동 패턴** | 아침에 할 일 목록 작성, 하루 중 완료 체크, 퇴근 전 진행률 확인 |
| **기대** | 가볍고 빠르게 할 일을 추가/완료 처리할 수 있는 도구 |
| **불만** | 기존 앱들은 기능이 너무 많거나, 계정 생성이 필요하거나, 느림 |

## 4. 기능 요구사항

### 4.1 핵심 기능 (P0 — Must Have)

| ID | 기능 | 설명 |
|----|------|------|
| F1 | 할 일 추가 | 텍스트 입력 + 카테고리 선택 후 추가. Enter 키 또는 버튼 클릭 |
| F2 | 할 일 수정 | 기존 할 일의 텍스트를 인라인 편집 |
| F3 | 할 일 삭제 | 개별 삭제 (확인 없이 즉시, 실수 방지를 위해 완료 항목만 일괄 삭제는 확인) |
| F4 | 완료 체크 | 체크박스 토글로 완료/미완료 전환. 완료 시 취소선 + 시각적 구분 |
| F5 | 카테고리 분류 | 3가지 카테고리: 업무(Work), 개인(Personal), 공부(Study) |
| F6 | 데이터 영속성 | localStorage에 JSON 형태로 저장. 모든 변경 시 자동 저장 |
| F7 | 진행률 표시 | 전체 및 카테고리별 완료율을 프로그레스 바로 표시 |

### 4.2 보조 기능 (P1 — Should Have)

| ID | 기능 | 설명 |
|----|------|------|
| F8 | 카테고리 필터 | 전체/업무/개인/공부 탭으로 필터링 |
| F9 | 완료 항목 숨기기 | 토글로 완료된 항목 표시/숨기기 |
| F10 | 완료 항목 일괄 삭제 | 완료된 할 일을 한 번에 정리 |

### 4.3 부가 기능 (P2 — Nice to Have)

| ID | 기능 | 설명 |
|----|------|------|
| F11 | 드래그 앤 드롭 정렬 | 할 일 순서를 드래그로 변경 |
| F12 | 다크/라이트 모드 | 사용자 선호 테마 전환 |

## 5. 데이터 모델

### 5.1 Todo 항목
```javascript
{
  id: "uuid",           // 고유 식별자 (crypto.randomUUID 또는 Date.now)
  text: "할 일 내용",    // 1~200자
  category: "work",     // "work" | "personal" | "study"
  completed: false,     // boolean
  createdAt: "ISO8601", // 생성 시각
  order: 0              // 정렬 순서
}
```

### 5.2 localStorage 구조
```javascript
// Key: "todo-manager-data"
// Value: JSON string
{
  todos: [ /* Todo 항목 배열 */ ],
  settings: {
    hideCompleted: false,
    activeFilter: "all"  // "all" | "work" | "personal" | "study"
  }
}
```

## 6. UI/UX 설계

### 6.1 레이아웃 구조
```
┌──────────────────────────────────────┐
│           Todo Manager               │
│     ─────────────────────────        │
│  [ 진행률 바: 8/15 완료 (53%) ]       │
│  [업무: 3/5] [개인: 2/4] [공부: 3/6]  │
│     ─────────────────────────        │
│  [전체] [업무] [개인] [공부]   [숨기기] │
│     ─────────────────────────        │
│  [ 할 일 입력...        ] [카테고리▼] │
│                            [추가]    │
│     ─────────────────────────        │
│  ☐ 회의 준비하기           🏢  ✏️ 🗑  │
│  ☑ 이메일 확인 ──────────  🏢  ✏️ 🗑  │
│  ☐ 운동하기               🏠  ✏️ 🗑  │
│  ☐ 알고리즘 문제 풀기      📚  ✏️ 🗑  │
│     ─────────────────────────        │
│  [완료 항목 일괄 삭제]                │
└──────────────────────────────────────┘
```

### 6.2 카테고리 시각 표현
| 카테고리 | 색상 | 아이콘 |
|----------|------|--------|
| 업무 (Work) | Blue (#3b82f6) | 🏢 |
| 개인 (Personal) | Green (#22c55e) | 🏠 |
| 공부 (Study) | Purple (#a855f7) | 📚 |

### 6.3 인터랙션
- **추가**: 입력 필드에 텍스트 입력 → 카테고리 선택 → Enter 또는 추가 버튼
- **완료**: 체크박스 클릭 → 즉시 토글 + 취소선 애니메이션
- **수정**: 텍스트 더블클릭 → 인라인 편집 모드 → Enter로 저장, Esc로 취소
- **삭제**: 삭제 버튼 클릭 → 즉시 삭제 (fade-out 애니메이션)

## 7. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **성능** | 100개 항목까지 버벅임 없이 렌더링 |
| **반응형** | 모바일 (320px) ~ 데스크톱 (1920px) 대응 |
| **접근성** | 키보드 탐색 가능, aria 레이블 적용 |
| **브라우저** | Chrome, Firefox, Safari, Edge 최신 버전 |
| **파일 구조** | 단일 HTML 파일 또는 HTML + CSS + JS 분리 (3파일) |

## 8. 기술 제약사항

- **프레임워크 금지**: React, Vue, Angular 등 사용하지 않음
- **빌드 도구 금지**: webpack, vite 등 없이 브라우저에서 직접 실행
- **서버 없음**: 정적 파일만으로 동작 (file:// 프로토콜에서도 동작)
- **외부 라이브러리 없음**: CDN 포함 외부 의존성 없음

## 9. 성공 지표

| 지표 | 목표 |
|------|------|
| 할 일 추가 → 화면 반영 | 100ms 이내 |
| 완료 토글 → 진행률 업데이트 | 즉시 (동기) |
| 페이지 새로고침 후 데이터 복원 | 100% |
| 첫 화면 로딩 | 500ms 이내 |
